Cola DLM

把"思考"和"造词"分开——先在潜空间里组织好整段意思，再让解码器一次性铺成文字
Continuous Latent Diffusion Language Model · Cola DLM

Hongcan Guo et al. · ByteDance Seed · arXiv 2605.06548 · 2026-05-08 · PDF · Project page

语言模型 扩散模型 潜变量 分层生成 2B 规模

一、Motivation

为什么这工作值得做。

具象痛点：让 GPT 写 100 字的产品介绍

你让 GPT 写一段"100 字以内的产品介绍"，它写到第 200 字还在加形容词，写到 500 字还想再加点意境。问题不是它不知道字数，是它没机会先想好整段要讲什么——每个 token 都是在前一个 token 的压力下落下来的，"全局结构"从来没真正存在过，只是在 token 链上以隐式的方式被勉强维持。AR 范式把"想"和"写"焊在一根链子上，链子越长，方向越歪。

这个痛点不是某个具体场景的问题，是所有 AR 模型共同的结构性短板：长输出收不住、infill 不顺手、局部编辑要重生成、几乎做不了非单调生成。背后是同一件事——"全局如何组织"和"局部怎么落字"被强行用一根 token 链表达。

前人卡在哪

过去几年里，三条路都没解开这个结：

  • AR 模型（GPT、LLaMA）给了最强的生成质量但锁死了左到右顺序；
  • 离散扩散（LLaDA、SEDD）打破了顺序但还在 token 空间里去噪——中间状态是"半 mask 的句子"，没有承载全局语义的能力，多步采样还慢；
  • 连续扩散在 token-aligned 空间（Plaid 等）把扩散用在 token 嵌入的恢复上——本质还是用扩散来恢复观测，不是用扩散来建模先验，连续性没用对地方。

关键的盲区在"扩散用来做什么"这件事上没想清楚。大家默认扩散是去噪工具，所以盯着"怎么把 token / token-embedding 去噪得更好"。没人停下来问：能不能换个层级——不去噪 token，而是去噪一个更抽象的语义状态？

作者的入口：搬运一个先验，而不是恢复一个序列

Cola DLM 的切入点很狠：把扩散从"观测恢复"换成"先验运输"。具体说，文本先经一个 Text VAE 压成连续潜变量 z₀，DiT 在潜空间里学这个 z₀ 的先验分布，最后 decoder 把 z₀ 解开成 token。扩散学的是"怎么从一团高斯噪声运到合法语义点"，不是"怎么从 noisy 文本恢复 clean 文本"。

这个换法的好处是把生成拆成了分明的两层——上层组织全局意思（潜变量先验），下层落实具体文字（解码器）。回到最开始那个产品介绍的例子：先在潜空间形成"100 字、活泼语气、强调耐用、结尾留 CTA"这样一个被压缩的整段意图；再让解码器一次性把这个意图铺成具体的字。意图先成形，字才有依归。

二、现存问题（前作的具体短板）

不是"已有方法效果不好"——是落到机制层面的具体卡点。

  • AR 链式分解：p(x) = ∏ p(xᵢ|x<ᵢ) 把所有语义压在 token 链里 → 强制左到右、生成顺序固定、不擅长 infilling/editing/non-monotonic 生成。
  • 离散扩散（LLaDA 类）：去噪还在 token 空间 → 中间状态是"半 mask 句子"无法表达连续语义结构 → 多步采样昂贵、全局规划弱。
  • Token-aligned 连续扩散（Plaid 类）：连续 path 走的是 token 嵌入的恢复轨迹 → 扩散在干"重建观测"的活，不是"建模先验"。继承了 AR 的缺陷，没拿到层次化的好处。
  • Token-embedding 扩散（DiffuSeq 类）：在嵌入空间扩散但缺乏分层潜变量解释，更像把噪声打回 clean embedding，不是真正分离"语义"和"实现"。
  • 已有 latent VAE-LM（Bowman 2016 等）：有潜变量但没强先验——通常假设 prior 是固定 Gaussian → 难以承载复杂语义结构、不能流式生成。

Cola DLM 的反诘是：这些路要么把扩散用错了层级（去噪 token 而非建模先验），要么有潜变量却没好先验。能不能两件事一起做？

三、相关工作翻译

这条路上前人都做过什么，免去你回去翻论文的工夫。

AR 语言模型（GPT / LLaMA / Qwen ...）

把文本写成 token 链，每一步预测下一个 token 的分布，用最大似然训练。简单、scaling 性能好、生态完整。代价是固定左到右、推理串行、长输出难收敛、不能并行块生成。这是当代 LLM 的默认范式。

LLaDA（离散扩散语言模型）

走"加噪—去噪"的扩散路子，但加的噪声是"mask 一些 token"，去噪是"猜回这些 token"。打破了 AR 的左到右束缚——生成顺序由 mask 模式决定。留下的问题：去噪空间还是离散 token，中间状态语义稀薄；采样要多步，每步都得跑大模型；难做长文本流式生成。Cola DLM 同意它"打破顺序"的目标，但不同意"在 token 空间去噪"的实现。

Plaid（连续扩散在 token-aligned 空间）

把 token 表示成连续向量 h = E(x)，然后在 h 上做扩散——加高斯噪声、训练去噪。本意是利用连续空间的几何来改善文本建模。留下的问题：扩散学的是"从 noisy h 恢复 clean h"，本质还是观测重建，不是先验建模。Cola DLM 把这个换成"把噪声运到 clean h"，连续性用在先验上而不是恢复路径上。

DiffuSeq / Diffusion-LM

在 token embedding 上做扩散，多用于 seq2seq 任务。机制上和 Plaid 类似——把 embedding 当 noisy target 来去噪。Cola DLM 认为这类工作缺乏明确的潜变量分层：embedding 既是 token 的代表又是扩散的目标，两个角色混在一起，没有"先验—解码"的明确分工。

早期 latent VAE 语言模型（Bowman 2016 等）

"VAE encode → continuous latent → VAE decode" 的祖师爷。引入了潜变量这个概念，但 prior 通常是固定的 N(0, I)，没足够表达力承载复杂语义结构。Cola DLM 继承这个分层思想，但把固定 prior 换成可学的扩散 prior，并加了 block-causal 结构支持流式生成、加了 BERT-style mask loss 强迫 latent 真正承担语义。

四、方法详解

两阶段训练 + 块式推理，用一个产品介绍的例子串起来。

整体架构（用产品介绍那个例子）

回到那个写产品介绍的场景。Cola DLM 在内部干这三件事：

  1. Text VAE 把"你写过的所有产品介绍"压成一片连续语义海——每段介绍变成 latent 空间里的一个点 z₀，相近语义的介绍点距离近。这一步在 Stage 1 干完。
  2. DiT 学会"怎么从噪声运到这片海里的合法点"——给定 prefix 条件（"100 字、活泼、强调耐用"），DiT 知道该把噪声运到哪个区域。这是 Stage 2 在干的事。
  3. 推理时：prefix → DiT 块式生成 latent → decoder 把 latent 一次性铺成具体文字。每个 latent block 对应 16 个 token，用 8-10 步去噪就能稳定生成。

Probabilistic 基础（公式一句话翻译）

整个生成模型由两个组件构成：先验 p_ψ(z₀) 和条件解码器 p_θ(x|z₀)：

$$p(x, z_0) = p_\theta(x \mid z_0)\, p_\psi(z_0), \qquad p(x) = \int p_\theta(x \mid z_0)\, p_\psi(z_0)\, dz_0 $$

先采一个潜变量 z₀（来自先验 p_ψ），再让解码器 p_θ 根据这个 z₀ 生成具体文本 x。和 Stable Diffusion 是同一思路——VAE 负责语义/像素分工，扩散学先验。

潜变量按 block 因式分解（Eq 3.3）：

$$p_\psi(z_0) = p_\psi\!\left(z_0^{(1)}\right) \prod_{b=2}^{B} p_\psi\!\left(z_0^{(b)} \mid z_0^{(\lt b)}\right)$$

latent 切成 B 个块，第一块无条件，后面每块条件于前面所有块。块内双向 attention 充分混合，块间因果 → 流式生成 + KV cache 像 AR 一样可缓存。

训练目标可以分解成三件事（Eq 3.5、关键的 ELBO 分解）：

$$\mathbb{E}_{p_\text{data}}[\mathcal{L}_\text{ELBO}] = \mathbb{E}_q[\log p_\theta(x \mid z_0)] - I_q(X; Z_0) - \mathrm{KL}(\bar{q}_\phi(z_0) \,\|\, p_\psi(z_0))$$

三项分别是重建质量（解码器多准）、压缩量（latent 携带多少信息）、先验拟合（学到的先验和实际后验的差距）。整个 Cola DLM 训练就是在这三项之间找平衡——这是论文最关键的拆解。

Stage 1：Text VAE 预训练

目标是建立"稳定的 text↔latent 映射"。Loss（Eq 3.16）：

$$\mathcal{L}_\text{VAE} = -\mathbb{E}_{q_\phi(z_0|x)}[\log p_\theta(x \mid z_0)] + \beta\,\mathrm{KL}\!\left(q_\phi(z_0|x)\,\|\,p_\text{base}(z_0)\right) + \lambda_\text{mask}\,\mathcal{L}_\text{mask}$$

三件事并列：(1) encoder→decoder 重建文本；(2) latent 后验贴近一个基础 prior（防止跑飞）；(3) BERT-style mask loss——把输入 token 随机 mask 后让模型用 latent 信号还原。第三项是防止"decoder 偷懒只记表面"的关键一刀。

为什么这么设计？ 没有 mask loss 时 decoder 可以靠 token-level pattern 直接 copy 输入；加了之后 latent 必须真正承担语义信号——实验里加 BERT loss 让 Tasks Avg 从 24 涨到 27（图 5）。

Stage 2：VAE + DiT 联合训练，DiT 学先验

从 Stage 1 的稳定 VAE 起步，把 DiT 拉进来一起训。先验学习用 Flow Matching（Eq 3.7）：

$$\mathcal{L}_\text{FM} = \sum_{b=1}^{B} \mathbb{E}_{t, z_0, z_1}\!\left[\left\|v_\psi\!\left(z_t^{(b)}, t;\, z_0^{(\lt b)}\right) - u_t^{(b)}(z_0, z_1)\right\|_2^2\right]$$

训练向量场 v_ψ，让它把高斯噪声 z₁ 沿着合法路径运到真实 latent z₀，每个 block 都看到前面 block 作条件。注意：Flow Matching 只是学先验的一种参数化，不是模型本身——换 DDPM、Rectified Flow 也行，核心定义是上面 Eq 3.1 那个分层结构。

Stage 2 总损失（Eq 3.18）四项：

$$\mathcal{L}_\text{stage2} = \lambda_\text{VAE}\!\left(-\mathbb{E}\log p_\theta(x|z_0) + \beta\,\mathbb{E}\log q_\phi(z_0|x) + \lambda_\text{mask}\mathcal{L}_\text{mask}\right) + \lambda_\text{fm}\mathcal{L}_\text{FM} + \lambda_\text{ref}\,\mathbb{E}\,\mathrm{KL}\!\left(q_\phi(z_0|x)\,\|\,q_{\phi_\text{ref}}(z_0|x)\right)$$

前一组保留 VAE 的自编码能力（不让 latent 退化），中间一项学 DiT 先验，最后一项用 reference encoder 锁住 latent 不要漂太远。第三项（reference KL）是关键——没它的话 VAE 和 DiT 联合训练会发散，论文实验里 All-Scratch 表现远差于 Joint-with-stable-init 就是这个原因。

Stage 3：推理（块式生成 + KV cache）

给定 prefix 文本 x_pre：

  1. 编码：z_pre ~ q_φ(z_pre | x_pre)
  2. 每个生成块 b：从噪声 ε^(b) ~ N(0,I) 出发，沿 v_ψ ODE 积分，条件于 prefix 和前面块：
    $$\tilde{z}_0^{(b)} = \Phi_{0\leftarrow 1}^\psi\!\left(\epsilon^{(b)};\, z^\text{pre},\, \tilde{z}_0^{(\lt b)}\right)$$ 块式因果保留了 KV-cache 的能力——block 间像 AR 那样缓存历史，block 内并行。
  3. 解码：x̂_res ~ p_θ(x_res | z_pre, z_0^(1:B))

为什么这条路真的更好？三个 governing curves

论文给了一个统一判据（Eq 3.35）——Cola DLM 比 AR 强当且仅当三件事同时成立：

  1. 低 rate 下的 D(R) 已经很小：数据真的存在低维全局语义结构，可压缩；
  2. 先验逼近误差 𝓔(M) 在缩小：DiT 学先验确实能学到位；
  3. 推理 gap G_infer 可控：变分近似不要崩。

三个条件缺一不可。这意味着分层结构本身不是 free lunch——只有当数据真的有可分层的结构时才赢。论文实验的 reasoning-heavy 任务（MMLU、RACE）上 Cola DLM 优势最大，证明这类任务的语义结构确实分层。

五、实验结果

不搬表格，提炼"什么动作带来什么收益"。

主结果一句话（RQ4）

在严格匹配的 ~2B 参数对比下（Cola DLM 含 500M VAE + 1.8B DiT；AR / LLaDA 同规模 backbone），Cola DLM 在 8 个 benchmark 平均上随 compute 持续上升、最终领先 AR 和 LLaDA，scaling 曲线最陡。优势集中在推理重 / 全局语义重的任务（MMLU、RACE、Story Cloze、OBQA），说明分层假设在这类数据上确实成立。生成型任务（LAMBDA、SQuAD）上和 AR 接近——因为这类任务对全局结构需求弱。

关键消融：什么动作能涨多少

VAE logSNR 设置（Tasks Avg @ 116.78 EFLOPs）

"可学的 logSNR" 比"固定 1.0"硬涨 +7.4 分——这是论文里最显著的单点收益之一。说明 VAE 的 logSNR（控制 latent 平滑度）必须随训练动态调整，不能拍脑袋固定。

Latent 维度对 Tasks Avg 的影响（all-scratch 设置）

维度从 16 → 128 涨了 +3.1 分；增量边际递减说明 latent 容量不是瓶颈，但维度还连带改变最优 timeshift（loc 从 1.0 漂到 2.3）——这是论文最 fascinating 的副发现：维度不仅放大空间，还重排了去噪轨迹的语义节奏。

第一块的 Conditioning 策略（Tasks Avg）

"clean condition repaint"（prompt 始终干净作引导）比 partial repaint 高 +15 分——这是论文里非常实用的工程教训：prompt 应该作为 stable conditioning context，不该自己也参与去噪。

其他关键发现（一句话总结）

  • Latent 空间策略：固定 VAE 早饱和，从零联训差，从稳定初始化 + DiT 联训（lr=1）是最强方案——Tasks Avg 比"固定 VAE"高约 1.5–2 分。
  • BERT mask loss + 主动更新：lr=1 + BERT loss 比"无 BERT loss"高 +2–3 分；BERT loss 在弱更新（lr=0.01）下提升有限——语义平滑性只有在 latent 主动进化时才发挥作用。
  • Block size = 16 是甜点：1 / 64 / 128 都更弱，"既不要太碎也不要太粗"。
  • Noise schedule loc=1.0 最优：尤其在 Joint DiT 设置下；uniform schedule 显著更差——调度参数其实在调"DiT 看到的有效语义信息曲线"。
  • 推理只要 8–10 步去噪已经能恢复绝大部分性能（饱和在 16–32 步）；配合 block size 16 = 比 AR 短 1.6–2× 的串行深度。
  • CFG ≈ 7 是甜点，太小（无引导）和太大（>10）都掉点，过强 CFG 会扭曲去噪轨迹。
  • Latent 压缩：patch size 2（两 token 一 latent）在 Mod0（偶长度）下平均 18.12，超过 patch=1 的 17.31；但 Mod1（奇长度）下崩到 0.26——问题不在压缩本身，而在边界对齐。
最反直觉的副发现 PPL ≠ 生成质量。Table 4 显示固定 VAE logSNR=1.0 时 token-level PPL 从 1.15×10⁶ 降到 641.57（"看起来好得多"），但实际生成的下个词从 "her" 变成了 "the"——困惑度大幅改善的同时生成质量在变差。论文给出 Implication 4：生成质量看的是潜空间的语义平滑度，PPL 看的是概率空间在 VAE logSNR 下的局部平滑度，两个"平滑"是两件事，所以可以朝相反方向走。

六、总结

  1. 核心贡献：把文本生成分层——上层在连续潜空间组织全局语义，下层用解码器实现具体文字；扩散在这套架构里负责"先验运输"而非"观测恢复"。这套分层为 LM 设计提供了一个新坐标系。
  2. 打开的新可能：因为潜空间是连续的，文本天然能和图像/视频共享同一个潜变量先验空间——指向一条 native 的统一多模态生成路径，不需要把图像硬塞进 token 序列。
  3. 留下的问题：(a) 当前只验到 2B 规模，更大尺度上分层优势是否继续保持未知；(b) PPL 与生成质量的偏离呼唤新评测；(c) 第一块的 mixed conditioning 还没有最优解；(d) latent 压缩遇到边界问题；(e) prior 的更强参数化（drifting model、AE 替代 VAE）值得继续探索。

七、Insight

论文里没直说，但逻辑指向的那颗结晶。

扩散的真正价值不是"去噪"，而是给你一个建模在合适抽象层级上的可分离先验。

过去几年里几乎所有"diffusion for text"的工作都在用一个强大的工具去干一个低层级的活——让扩散去恢复 token、去恢复 token embedding、去恢复 noisy 文本。它们没问的问题是：扩散应该作用在哪一层？Cola DLM 给的回答是：作用在压缩后的语义层，让"语义形成"和"文字落地"分开成两件事。这一刀切下去，扩散从"纠错器"变成"先验生成器"，重要性从战术变成战略。

这个洞见可以推广：当数据存在两层结构（语义 + 表面）时，用同一根链子建模两层是浪费。AR 之所以管用，是因为文本表面恰好是承载语义的强 scaffold——但这是个偶然，不是必然。Cola DLM 暗示一种未来：scaffold 可以是潜变量，token 退化成"潜变量的 surface"，而不是"语义本身"。

另一个值得抽出来贴墙上的发现：困惑度和生成质量度量的是不同的"光滑"。一个模型架构变了之后，沿用旧度量是危险的——你可能因为数字"看起来在改善"而把模型推向实际能力变差的方向。Cola DLM 用 Implication 4 把这件事从经验观察上升到机制解释：当模型显式分层，PPL 测量的"概率空间局部平滑"和生成测量的"语义空间全局可达性"会朝不同方向变化。

八、启发（对我能用的提醒）

落点是"你可以___"，不是"值得思考___"。

迁移：把分层用在我自己手头的任务上

Cola DLM 的"先压成意图 latent，再展开 token"模式可以直接搬到任何"全局规划 + 局部实现"的任务：多步推理（plan latent → reasoning chain）、长文档生成（structure latent → 段落实现）、代码生成（function-level intent latent → token-level code）。明天可以试的小动作：选一个 long-form 生成任务（如多步规划），在现有 AR 模型外面套一个 latent encoder，先生成 plan latent，再把 plan latent 作 prefix 给 decoder——看 plan latent 是否真的提高了规划一致性。

混搭：把它和现有工具组合

Cola DLM 的块因果潜变量和CoT / scratchpad reasoning是天然 pair：让 latent 块承载"思路 sketch"（不出字），decoder 把 latent 块铺成可读 reasoning steps。这能解决一个老问题——CoT 把"内部思考"暴露成 token 强迫所有思考都得是文字，浪费表达带宽。明天可以试：在数学题上用 latent thinking + token answering 的两段式，对比纯 CoT 的 token efficiency 和准确率。

反转：停用 PPL 作为唯一信号

Cola DLM 的 PPL ≠ generation 结果颠覆了一个默认假设。停下来：不要再把 PPL 改善当作"模型变强"的硬证据，尤其是对于任何含显式潜变量、含 mask loss、含 latent regularization 的模型。开始：把多次 generation 的 task accuracy、matched-compute scaling 曲线作为主要评测信号，PPL 退化为辅助监控指标（追踪训练稳定性而非模型能力）。

九、关键引用（双语对照）

脱离论文上下文还能记一年的句子。

Abstract · 论文宗旨

"Cola DLM, a hierarchical latent diffusion language model that frames text generation through hierarchical information decomposition... its diffusion process performs latent prior transport rather than token-level observation recovery, thereby separating global semantic organization from local textual realization."

Cola DLM 把文本生成拆成"先组织全局语义、再实现局部文字"两步——它的扩散过程是搬运一个潜变量先验，不是去恢复 token 序列。

Section 3.3 · 连续性该用在哪

"The reason for using a continuous path is not that continuous modeling is inherently superior, but that it naturally captures the geometry of the latent distribution. In Cola DLM, continuity appears in p_ψ(z₀), rather than in an observation-recovery trajectory."

用连续路径不是因为连续比离散好，而是因为连续天然适合潜变量分布的几何。Cola DLM 的连续性是用在"先验本身"上，不是用在"恢复输入序列"上。

Section 5.1 · Implication 4 · PPL 不等于生成质量

"In Cola DLM, generation quality is more related to semantic smoothness of the latent space, whereas likelihood-oriented PPL is more sensitive to probability-space smoothness shaped by the VAE logSNR. Since these two forms of smoothness are different, generation and PPL need not be aligned."

在 Cola DLM 里，生成质量看的是潜空间在语义上够不够顺滑，而 PPL 看的是概率空间在 VAE logSNR 调节下够不够顺滑——这两个"顺滑"是两件事，所以生成好不见得 PPL 就低。

Section 4.3 · 潜空间该如何进化

"The most effective strategy is to let it evolve together with DiT on top of a stable initialization."

让潜空间从一个稳定的初始化出发、跟 DiT 一起更新——这是最好的策略。既不能完全冻死，也不能从零乱训。

Section 6 · 框架真正的价值

"The main value of Cola DLM lies not merely in the denoising process itself, but in its decomposition of text generation into global semantic prior modeling and local textual realization."

Cola DLM 的真正价值不在于"去噪"这个动作本身，而在于它把"语义组织"和"文字实现"拆成了两个层级。

十、Q&A

12 道面试题，覆盖结构、输入输出、数据、训练流程、策略、具体方法、实验细节、结果思考与延伸。先想再展开。

结构Q1：Cola DLM 整体由哪几个模块组成？模块之间怎么连？
三大模块，全部基于 transformer：
  • Text VAE Encoder q_φ(z₀|x) — 把文本 token 序列映射成连续 latent z₀ ∈ R^(L×d)（d=16 是论文甜点）。约 500M 参数。
  • Block-causal DiT p_ψ(z₀) — 在 latent 空间学先验。"block-causal" = 块内 bidirectional，块间 causal：每个块（默认 16 个 latent token）内部双向 attention 充分混合局部信息，块间保留因果依赖以支持 KV-cache 流式生成。约 1.8B 参数。
  • Text VAE Decoder p_θ(x|z₀) — 把 latent 解码回 token 序列。和 encoder 共享 500M。
连接方式：训练时 encoder → latent → DiT 学先验 + decoder 重建（联合优化）。推理时 encoder 编 prefix → DiT 块式生成续写 latent → decoder 一次性铺成 token。块因果是关键设计——既保留了 transformer 的并行优势，又支持 AR 般的流式 KV-cache。
输入输出Q2：训练时和推理时的输入输出形态分别是什么？维度怎么对应？
训练时：
  • 输入：文本 token 序列 x ∈ V^L_token（L_token ≤ 512）
  • 中间表示：latent z₀ ∈ R^(L_latent × d)，d=16，L_latent 取决于 patch size——p1 时 L_latent = L_token，p2 时 L_latent = L_token/2（每两 token 压成一个 latent）
  • 输出：Stage 1 是重建文本 + 损失；Stage 2 是联合损失（VAE 重建 + Flow Matching 先验 + reference KL）
推理时：
  • 输入：prefix 文本 x^pre
  • 编码：z^pre ~ q_φ(z|x^pre)
  • 逐块生成：每块 16 个 latent token，从 N(0,I) 出发用 8–10 步 ODE 积分到 z₀^(b)，条件于 prefix 和前面的块
  • 输出：所有块生成完后 decoder 一次性解出文本 token
关键：输入端是离散 token，中间是连续 latent，输出端又是离散 token——这正是 Cola DLM 的"分层"在维度上的体现，连续性只发生在中间这一段。
数据Q3：用什么数据训练？规模和评测协议有什么特别之处？
训练数据：外部公开预训练语料（论文未指定具体来源，等量级常见为 RedPajama、SlimPajama、Dolma 等）。Tokenizer 使用 OLMo 2。最大序列长度 512。所有方法（Cola DLM / AR / LLaDA）共享相同的 training data + random seed，确保对比公平。

评测协议特别之处：这是论文最 thoughtful 的设计之一——统一的 few-shot generative evaluation，跨多选题和生成题都用同一种方式评测。具体：
  • 多选题（MMLU、SIQA、OBQA、HellaSwag、RACE、Story Cloze）也用生成方式评，让模型生成完整答案再 string-match，而不是用 likelihood 选最大选项
  • 生成题（LAMBADA、SQuAD）正常生成评测
为什么这么做：因为 Cola DLM 的 PPL 与生成质量本身就脱钩（见 Implication 4），用 likelihood-based 多选会把 Cola DLM 评得很惨但与实际生成能力不符。统一生成评测才能让分层模型和 AR 在同一根尺子上比。这是评测方法学上一个值得记住的点：评测协议要跟模型的训练目标对齐。
流程Q4：训练分几个阶段？每阶段做什么？为什么必须分？
两阶段。

Stage 1（Text VAE 预训练）：只训 encoder + decoder。损失 = 重建 + KL（贴一个 base prior p_base）+ BERT-style mask loss。目标是建立稳定的 text↔latent 映射。这阶段的 prior 是固定的辅助 regularizer，不是最终模型的 prior。

Stage 2（VAE + DiT 联合训练）：从 Stage 1 出发，把 DiT 拉进来一起训。损失 = VAE 重建（继续约束 latent-text 关系）+ Flow Matching 先验学习（DiT 学真正的 prior）+ Reference KL（用 frozen Stage 1 encoder 锁住 latent 不漂走）。

为什么必须分：实验图 3 / 4 直接显示——All-Scratch（不分阶段，VAE+DiT 从零联训）显著差于 Stage1-then-joint。原因机理：DiT 学先验需要一个语义结构已组织好的 latent 起点，否则在追一个不停剧变的目标，latent 几何会崩塌（图 4 显示 All-Scratch d=16 的 latent self-similarity 矩阵几乎是均匀色块——意味着 latent 完全没分化）。Stage 1 提供这个稳定地基，Stage 2 在地基上让 latent 与 DiT 协同进化。
策略Q5：训练里有哪些关键策略？每个解决什么具体问题？
四个能记住的关键策略，每个都解决一个具体毛病：
  1. BERT-style mask loss（Stage 1+2）— 解决"decoder 偷懒只记表面"。把输入 token 随机 mask 后让模型用 latent 信号还原，强迫 latent 真正承担语义。+2–3 分（图 5），且只在 latent 主动更新（lr=1）时显著。
  2. Reference KL（Stage 2 第三项）— 解决"联训发散"。用 frozen 的 Stage 1 encoder 作 reference，约束当前 encoder 不要漂得太远。没它的话 VAE+DiT 联训会让 latent 飞走。
  3. VAE-DiT 学习率比 = 1（Stage 2）— 解决"latent 进化不够 vs 进化太狂"的取舍。lr=0.01 时 latent 几乎不动（co-evolution 失败），lr=1 时 latent 主动随 DiT 进化（结合 reference KL 防漂走）。这是论文最反直觉的发现之一——主动联训远胜于轻微调整。
  4. Learnable VAE logSNR（控制 latent 分布的方差）— 解决"latent 平滑度该多少"的取舍。固定值（1.0 / 1.5 / 2.0）都不如学的（学到 ≈4.5）。固定 1.0 vs 学的 = 14.7 vs 22.1，差 +7 分，是论文最显著的单点收益。
四条加起来勾勒出一种"既约束又放手"的策略哲学：BERT 约束 latent 携带语义、reference KL 约束不漂走、lr=1 放手让 latent 随 DiT 进化、学 logSNR 让平滑度自适应。
方法Q6：怎么直觉地理解"扩散用于先验运输而不是观测恢复"？
用一个类比：
  • 传统扩散 LM（Plaid、DiffuSeq 等）= 修图匠。你给它一张被加噪的脸（noisy embedding），它一步步还原成清晰的脸（clean embedding）。目标是恢复一个已存在的观测。
  • Cola DLM = 画家。你给它一张白纸（高斯噪声），它一笔笔运笔画出一幅作品（latent z₀）。然后 decoder 把这幅画"翻译"成具体文字。目标是建模一个新的生成分布。
区别的本质：修图思路里扩散是工具人，模型核心还在 token 空间打转，没有显式的"先组织语义"层级。画家思路把 latent prior 显式分离——扩散运输到合法语义点 → 解码器铺成具体形式。

关键转换：扩散从"恢复观测的工具"升级为"建模先验的引擎"。这一升级让分层结构落地——上层组织语义，下层实现表达。Cola DLM 的所有性能优势都源自这个升级，不源自"连续比离散好"或"扩散比 AR 好"这种表面判断。
方法Q7：用 Flow Matching 而不是 DDPM 是什么考量？为什么作者强调"FM 只是实现选择"？
模型的核心定义是分层结构 p(x) = ∫ p_θ(x|z₀) p_ψ(z₀) dz₀。p_ψ 是个连续 prior，怎么参数化都行。

选 Flow Matching 的工程理由：FM 训练向量场 v_ψ 直接拟合"高斯噪声 → 真实 latent"的运输路径，比 score matching 训练目标更稳定、时序更可控、和 ODE 求解器配合更好。

"FM 只是实现选择"是什么意思：作者刻意不让模型 identity 被"扩散参数化"绑死。理论上，DDPM、Rectified Flow、EDM、甚至非扩散的 normalizing flow 都能学这个 prior，分层结构本身不变。这种解耦让框架可移植——别人可以拿走分层骨架，装上自己喜欢的扩散参数化。

这一点对方法学很重要：很多论文把"用了 X 技术"和"模型本质"混在一起，导致后人无法判断哪部分是 fundamental、哪部分是实现细节。Cola DLM 这种解耦写法是好范例。
实验Q8：评测哪些 benchmark？baseline 怎么选？复现要注意什么？
评测集（共 8 个）：
  • 内部消融（RQ1–3）：LAMBADA、MMLU、SIQA
  • 外部 scaling 对比（RQ4）：再加 SQuAD、Story Cloze、OBQA、RACE、HellaSwag
Baseline：
  • AR：LLaMA-style，按官方实现随机初始化重新训
  • LLaDA：discrete diffusion LM，按官方实现重新训，inference denoising length 等于生成长度
  • 严格匹配条件：non-embedding backbone 都是 1.8B、共享 OLMo 2 tokenizer、共享训练数据 + random seed、max length=512
复现要注意的坑：
  1. 必须用统一 few-shot generative evaluation，不要用 likelihood-based MCQ（Cola DLM 的 PPL 不可靠）
  2. Stage 1 → Stage 2 之间不要重置 VAE 权重——All-Scratch 会显著掉点
  3. 必须加 reference KL 项，否则 Stage 2 联训发散
  4. VAE logSNR 必须设为 learnable，不要固定
  5. DiT block size = 16 是甜点；1 / 64 / 128 都更弱
  6. Inference 用 8–10 步去噪 + CFG=7 即可，不必跑到 32 步
思考Q9：Table 4 / Implication 4 的"PPL 改善但生成变差"——本质是什么？对评测领域意味着什么？
本质：PPL 和生成质量量的是不同维度的"光滑"。
  • PPL 量的是posterior 邻域的概率密度——围绕 ground-truth token 的局部光滑度
  • 生成质量量的是prior 落到 decoder-valid 区域的能力——latent 空间的全局可达性
当模型显式分层（VAE + diffusion prior），这两个光滑可以朝相反方向走。压低 VAE logSNR 会让概率密度更光滑（PPL 改善），但代价是模糊语义边界（生成偏向 the / on / in 这种通用空洞词，反而看不见 her / settled 这类承担含义的词）。Cola DLM 的 Table 4 + Implication 4 就把这件事从经验观察上升到了机制解释。

对评测领域的意味：
  1. 所有含显式潜变量结构的模型都应停用 PPL 作为主要评测信号——VAE-LM、latent diffusion LM、hierarchical LM。AR-LM 时代沿用的"PPL 改善 = 模型变强"是个特化结论，不通用。
  2. 评测协议要跟训练目标对齐。AR 训练 -log p(x|x_
  3. 替代信号：matched-compute scaling 曲线 + task-level generation accuracy。这是 Cola DLM 在 Section 4.5 用的协议。
这条对所有"超出 AR"的语言模型研究都适用——别再被 PPL 数字骗了。
思考Q10：Figure 10 显示 Cola DLM 在 reasoning-heavy 任务（MMLU/RACE/Story Cloze/OBQA）上优势最大，generative 任务（LAMBADA/SQuAD）上和 AR 接近。为什么？告诉我们什么？
为什么：
  • Reasoning-heavy 任务依赖跨段语义整合——选项之间的关系、故事的因果链、世界知识的远距离 binding。Cola DLM 的 latent 显式建模这层全局结构（潜变量 z₀ 一次承载整段意图），AR 把它隐式压在 token chain 里。需要跨段整合时，显式优于隐式。
  • Generative 任务对全局结构需求弱（短上下文、局部续写），AR 的 token-level 拟合就够，分层带来的优势小。
告诉我们什么：
  1. 分层结构不是 free lunch——优势仅在数据真有可分层结构时显现。这从反面验证了论文判据 Eq 3.35（D(R) 在低 R 已经够小）。
  2. 选 task 来 showcase 分层结构时要选有强全局结构的——长上下文推理、跨段问答、世界知识 binding。short-context generation 不是好的 showcase 场景。
  3. 方法和任务匹配是个常被忽略的研究方向——不是所有任务都需要分层，但需要的任务上分层有结构性优势。
延伸Q11：Cola DLM 最关键的未被深究的预设是什么？如果它不成立会怎样？
最关键的预设："文本数据真的存在低维 + 可分离的全局语义结构"——即 Eq 3.35 的第一个条件 D(R) 在低 R 已经够小。

如果不成立，崩在哪：极度依赖局部精确表达的领域（代码 token 级、法律条文、强格式输出、化学式、数学推导），强行压到低维 latent 会丢必要信息。崩点在Stage 1 的重建项：
  • 要么 VAE 怎么训都过不了 reconstruction loss → 模型不收敛
  • 要么 latent 维度被迫爆炸到接近 token-level（失去压缩意义）
  • 要么 decoder 退化回直接 copy 输入（latent 没用，整个分层结构等于没建）
论文未深入讨论的点：不同领域的"语义可分层度"差距可能很大。论文只在通用文本上验证。这是后续工作最值得攻击的方向——比如代码 LM 是不是该用更高维 latent？医疗记录 LM 是不是该 patch size = 1？目前没人系统研究过。
延伸Q12：这套"分层+扩散先验"的思想能反向用在 image diffusion 上吗？怎么用？
实际上 Stable Diffusion + ControlNet 已经在做部分这件事：先有 layout / pose / depth 的 semantic plan，再用 diffusion 生成像素。但 plan 是手工指定的或外部模型给的，不是学到的 prior。

Cola DLM 的进一步启发：把这个 plan 也用一个学到的 prior 建模——给 image diffusion 加一个 "plan latent" 层，由另一个 transport 模型（也可以是 DiT）把噪声运到合法 plan latent，然后 SDXL 拿这个 plan latent 作 condition 生成像素。具体形态： 
text → plan-latent (DiT₁) → image-latent (SDXL VAE) → image (SDXL Decoder)
相当于把 SD 的 text encoder 替换成"text → plan latent 扩散运输"——和 Latent Consistency / 可控生成的工作天然合流。

更激进的想法：plan latent 可以共享于 text 和 image。同一个潜空间里同时生成"故事板的描述"和"故事板的图像"——这就是论文 Section 5.5 暗示的 native 多模态路径，不需要把图像硬塞进 token 序列。这条路如果走通，会改变多模态生成的范式。

明天可以试：在 SDXL 的 cross-attention 之前加一个轻量 DiT 把 text embedding 运到 plan latent，看是否能改善"text-image alignment"或"长 prompt 一致性"。
