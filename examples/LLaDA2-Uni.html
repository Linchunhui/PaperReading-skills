LLaDA2-Uni 阅读笔记

看懂即能画——用同一套"语义离散码"统一多模态理解与生成
LLaDA2.0-Uni: Unifying Multimodal Understanding and Generation with Diffusion Large Language Model

Inclusion AI · arXiv 2604.20796 · 2026-04-22 · PDF · GitHub · HuggingFace

多模态 离散扩散 统一理解+生成 MoE 16B SigLIP-VQ

摘要

这是一篇把"看懂图"和"画图"塞进同一个离散扩散语言模型的工作。痛点是：当下的"统一多模态模型"要么用 AR 范式（Janus、BAGEL），要么用纯离散扩散（MMaDA、Lumina-DiMOO），但后者一直跟 AR 路线在 understanding 上有明显差距——根因是它们用的 VQ token 是像素重建型，丢了语义。

作者的招数：把 VQ tokenizer 换成"语义型"的 SigLIP-VQ（基于 SigLIP2 ViT 的特征做向量量化，码本大小 16384、维度 2048），让同一套离散 token 既承担理解又承担生成。整个架构三件套——SigLIP-VQ tokenizer + 16B MoE 离散扩散主干 LLaDA2.0-mini + 6B Z-Image-Base 扩散解码器，端到端用同一个 block-wise mask prediction 目标训练。

关键发现：在 21 个理解 benchmark 上追平专家级 VLM（Qwen2.5-VL-7B），在 GenEval / DPG-Bench / UniGenBench 上大幅领先所有"统一模型"对手（GenEval 0.89，UniGenBench 79.63），编辑能力（MICo-Bench 47.1）也成了新 SOTA。再加两个免训练加速：SPRINT 给推理 1.6× 加速，蒸馏 decoder 把生成提速 11.4×（50 步 → 8 步）几乎无损。有意思的是：开"思考模式"在 WISE-Bench 上能再涨 +10 分——离散扩散框架天然支持 interleaved reasoning。

一、Motivation

为什么这工作值得做。

具象痛点：你想要一个"既能看懂图、又能画图"的模型

设想这个场景：你给模型一张图，问它「画里哪个区域降水最高？」——它告诉你"南部，5 月，约 200 mm"。紧接着你说「再帮我把黑板上的字改成 LLaDA Coffee」——它用同一个模型把图片改了。再下一步「请生成一个泡澡的小熊插画」——它继续用同一个模型画出来。

这就是统一多模态模型的承诺：理解和生成两件事不再是两个模型、两个 pipeline、两套 token，而是一个 backbone 上的两个 mode。好处明显：互相增强（理解越好 prompt 越准、生成越好图像表达越细）、部署简单、能解锁 interleaved 生成（一段文字 → 一张图 → 一段文字 → 又一张图，像聊天那样自然交错）。

前人卡在哪

当前两条路都没做漂亮：

  • AR 派（Janus、BAGEL、OmniGen2、HunyuanImage 3.0、Z-Image）— 把图像离散化成 token 然后 next-token predict。能 work，但需要在 backbone 上微妙地平衡 AR loss 和 diffusion loss，工程上复杂；并行解码的优势也部分丢失。
  • 纯离散扩散派（MMaDA、Lumina-DiMOO）— 用 mask-and-recover 的离散扩散统一两个任务，理论很优雅：单一目标、并行解码、双向上下文。但实际表现一直追不上 AR——根本原因不在扩散本身，在它们用的VQ-VAE token 是为像素重建训的，里面没语义信息，模型拿这种 token 去做 understanding 就力不从心。

其他方案比如 LLaDA-o、BAGEL 用双视觉模块（ViT 编码用于理解、VAE 编码用于生成）来回避这个问题——但又制造了新的"modeling gap"：同一个 backbone 内部要服务两套异构的视觉表示。

作者的入口：把 VQ token 从"重建型"换成"语义型"

作者的判断很直接：不是离散扩散这条路不行，是这条路上用的 token 不行。如果 VQ token 本身就承载语义（基于 SigLIP 这种 vision-language 对齐特征做量化），那它既适合 understanding 又适合 generation——纯离散扩散统一就立得住了。生成时用一个专门的 diffusion decoder 把语义 token 翻回像素就行。

整篇文章的所有设计都围绕这个核心判断展开：SigLIP-VQ tokenizer + LLaDA2.0-mini 16B MoE backbone + Z-Image-Base 6B 扩散解码器，端到端一个 BDLM (Block Diffusion Language Model) 目标训。回到刚才的场景——读图、改图、生成图，全部走同一个 backbone、同一种 token、同一个目标函数。

二、现存问题（前作的具体短板）

不是泛泛说"效果不好"——落到机制层面的具体卡点。

  • 问题 A：重建型 VQ token 没语义 → MMaDA、Lumina-DiMOO 用 VQ-VAE，token 是为"还原像素"训的，里面没承载"这是什么"的信息 → understanding benchmark 上比 AR-VLM 低十几个点（OCRBench 上 Lumina-DiMOO 7.6 vs Qwen2.5-VL-7B 84.2，差 11×）
  • 问题 B：过度图像压缩损生成质量 → VQ-VAE 一般压缩比很高（每 16×16 像素一个 token），细节丢失明显 → 高保真生成、文字渲染、人体细节这些 detail-sensitive 任务直接崩
  • 问题 C：纯双向 attention 训练不稳 → 扩散 LM 理论上应该用 full bidirectional attention，但实测会 degrade performance，跟 SigLIP-VQ token 自带的 AR bias（来自 SigLIP2 跟 Qwen2.5 对齐过）冲突
  • 问题 D：双视觉模块的 modeling gap → BAGEL、LLaDA-o 用 ViT 做理解 + VAE 做生成 → 两套表示在同一个 backbone 里，优化目标分裂，像让一个学生同时学两种相互冲突的"看图方法"
  • 问题 E：固定输出长度限制开放场景 → 现有离散扩散 LM 假设 understanding 输出长度固定（提前预定 mask token 数），但实际 VQA / chart QA 答案长度差异巨大 → 短的浪费、长的截断

三、相关工作翻译

这条路上前人都做过什么，免去你回去翻论文的工夫。

Janus / Janus-Pro（2025，DeepSeek）

AR 派统一模型的代表。把图像 tokenize 成离散序列，理解和生成共用一个 transformer，但用两个独立的 vision encoder——理解走 SigLIP，生成走 VQ-VAE。绕开了"一种 token 难兼顾"的问题，代价是模型内部有两套异构视觉表示，没真正统一。

BAGEL（2025，ByteDance）

"AR + Diffusion 混合"路线。在一个 transformer 里同时跑 next-token prediction（理解）和 diffusion loss（生成）。性能很强，但训练工程复杂——两个 loss 的权重平衡、两条梯度路径打架。

OmniGen2 / HunyuanImage 3.0 / NextFlow

都是"AR + 扩散"思路的变种。差别在于扩散是用作 image generation head（OmniGen2、Hunyuan）还是更深度地整合（NextFlow）。共同问题：架构非平凡、训练 pipeline 复杂、loss 平衡敏感。

MMaDA / Lumina-DiMOO（2025，纯离散扩散派）

真正纯粹的离散扩散统一模型——单一 mask prediction 目标、单一 token 体系。理论最优雅，但用的是重建型 VQ-VAE。结果：generation 还行，understanding benchmark 上离 AR 派 SOTA 有十几分差距，OCR/document 这类细粒度理解几乎做不了。这是 LLaDA2.0-Uni 直接的"battlefield"——证明纯离散扩散这条路没问题，问题在 VQ token 选错了。

LLaDA-o（2026，同团队前作）

同样是离散扩散统一模型，但仍用双视觉模块（ViT 理解 + VAE 生成）。性能比 MMaDA / Lumina-DiMOO 强，但内部仍有 modeling gap。LLaDA2.0-Uni 是它的进化——把 dual encoder 合并成单一 SigLIP-VQ，架构更纯净。

四、方法详解

三件套架构 + BDLM 训练 + 两个免训练加速。回到读图/改图/画图的例子串起来讲。

组件 1：SigLIP-VQ Tokenizer（核心创新）

做什么：把任意分辨率的图像编成一串离散语义 token（码本大小 16,384，每个 code 维度 2048）。

怎么做：基于预训练好的 SigLIP2-g ViT（vision-language 对齐过的视觉编码器）作为 feature extractor，后接一个 vector quantizer 做离散化。这套架构来自 X-Omni（Geng et al., 2025）。关键不同于传统 VQ-VAE：传统 VQ 在像素重建任务上训，token 学的是"如何精确还原 RGB"；SigLIP-VQ 在 understanding 任务上训，token 学的是"这是什么"——语义信息直接编进去。

为什么这么做：回到刚才的"读图问降水"场景——SigLIP-VQ 编出的 token 直接带"南部 / 月份 / 数值"这类语义维度，模型读这串 token 就能理解；而传统 VQ token 编的是"红色像素 / 折线斜率"，模型还得自己再学一层语义抽取，效率低且准确率差。缺点：SigLIP-VQ 没有原生的像素 decoder，需要额外的 diffusion decoder 来把 token 还原成图（组件 3）。

组件 2：16B MoE 离散扩散主干（LLaDA2.0-mini）

做什么：处理混合的 text/image token 序列，预测被 mask 掉的 token。

怎么做：基础是 LLaDA2.0-mini（一个 16B 参数的 modality-agnostic MoE 扩散 LM）。改造点：扩展原 LLM 词表，加入 SigLIP-VQ 的 16,384 个视觉 token + 若干特殊控制 token；input embedding 层保留原文本 embedding，新视觉 token 随机初始化；输出 head 同步扩展，文本部分继承预训练。

三个关键设计：

  1. Block-wise Attention（来自 Arriola et al., 2025）— 块内 bidirectional，块间 causal。理论上扩散 LM 该用 full bidirectional，但实测 degrade。Block-wise 是兼顾"块内并行 + 不打乱 SigLIP-VQ token 自带的 AR bias"的折中。
  2. Modality-agnostic MoE — MoE 让 backbone 充当"通用多任务学习器"，按 input 动态分配 expert，无需为每种 modality 设计专门子网。
  3. 1D RoPE + size token — 不像很多多模态模型用 2D RoPE 处理图像，作者保持 1D RoPE，仅在视觉 token 序列前加 <height_512>、<width_512> 这样的 size token。这个简化方法已被 Liu et al. 2026、Geng et al. 2025、Xin et al. 2025b 验证有效。好处：原生支持任意分辨率、无需架构改动。

组件 3：扩散解码器（Z-Image-Base 6B）

做什么：把 backbone 输出的 SigLIP-VQ token 翻译回真实像素图像。

怎么做：基于 Z-Image-Base（一个 6B 文生图扩散模型）。但不再用文本作为条件，而是把 SigLIP-VQ 语义 token 作为唯一 conditioning signal（替换原来的文本 prompt 嵌入）；同时 decoder 还做 2× 超分（用 upsampled semantic token 做 condition 直接生成更高分辨率）。这与 NextFlow、X-Omni 不同——它们冗余地把文本 prompt 和视觉 token 都喂给 decoder。

为什么这么做：语义 token 已经承载了 prompt 信息（backbone 看完用户 prompt 才生成的这些 token），再喂一遍文本是冗余还可能分流注意力。直接信任 token、专心做 token→pixel 翻译，反而干净。

训练目标：BDLM Loss

核心训练目标是 Block Diffusion Language Model loss（来自 Arriola et al., 2025）：

$$\mathcal{L}_{\text{BDLM}}(\theta) = -\,\mathbb{E}_{t, \boldsymbol{x}_0, \boldsymbol{x}_t}\!\left[ \frac{\alpha'_t}{1-\alpha_t} \sum_{k=1}^{K} \sum_{i=1}^{L_B} \mathbb{1}[x^i_{t,k} = \text{[MASK]}] \cdot \log p_\theta(x^i_{0,k} \mid \boldsymbol{x}_{0,\lt k},\, \boldsymbol{x}_{t,k}) \right]$$

把序列切成 K 个 block，每块长 L_B；按 timestep t 给每块 mask 一部分 token，模型负责预测这些 mask 位置的原 token。前缀 x_{0,<k}（已生成的 clean block）作为条件，当前块 x_{t,k} 是部分被 mask 的版本。系数 α'_t/(1-α_t) 是扩散 schedule 给每个 timestep 加的权重。

SFT 阶段：Mask Token Reweighting (MTRS) Loss

SFT 时多了个挑战——sample 长度差异巨大（最多 100×）。token-averaged loss 会被长 sample 主导，sample-averaged 又鼓励短输出。论文设计了一个折中：

$$\mathcal{L}_{\text{MTRS}} = \frac{\sum_j \beta_j\, \mathcal{L}_{\text{SFT}}^{(j)}}{\sum_j \beta_j}, \quad \text{where} \quad \beta_j = \frac{1}{\sqrt{\sum_{k,i} \mathbb{1}[x_{t,k}^{i,(j)} = \text{[MASK]}]}}$$

每个 sample 的权重 β_j 是它 mask token 数量的平方根倒数——长 sample 权重小、短 sample 权重大，但不是 sample-averaged 那样的极端。这个 1/√N 的设计平衡了"梯度不被长样本主导"和"不偏向短输出"两个矛盾。

SPRINT：免训练推理加速（1.6×）

SPRINT = Sparse Prefix Retention with Inference-time Non-uniform Token Unmasking。两个正交维度的优化，组合起来 1.6× 加速、性能损失仅 -0.6 分。

(a) Sparse Prefix Retention（稀疏前缀保留）：每步去噪都要 attend 到完整 prefix，attention cost 是二次方。SPRINT 在每个 block 第一步做完整 forward 后，用一个综合 importance score 给 prefix 每个位置打分：

$$s_i = \alpha \cdot \tilde{l}_i + (1 - \alpha) \cdot c_i$$

l̃_i 是这个位置 key 的归一化范数（反映它对 attention 的影响程度），c_i 是这个位置预测的 top-1 confidence（反映模型对它的把握），α=0.5。关键是 modality-aware——文本 token 保留率 r_text=1.0（不剪），图像 token 保留率 r_img=0.8（剪 20%）。理由：图像 token 空间冗余高、可剪；文本 token 承载指令和推理链、不能剪。

(b) Non-uniform Token Unmasking（非均匀去 mask）：标准 schedule 每步固定去 mask 一定比例的 token，浪费在已经高置信的 token 上。SPRINT 改成 confidence-adaptive：

$$\mathcal{A} = \{n \in [m] : c_n > \tau\}$$

所有 confidence 超过阈值 τ（取 0.93 或 0.95）的 mask 位置都一次性解开。每步至少解 ⌈m/(T-t)⌉ 个保证终止。结果：高置信 token 提前解决，难 token 多分几步——把"算力预算"花在真正需要的 token 上。

扩散解码器蒸馏（11.4× 加速）

原 decoder 50 步采样要 32.95 s/img，蒸馏到 8 步只要 2.90 s/img——加速 11.4×。蒸馏目标结合 flow matching + consistency loss：

$$\mathcal{L}_{\text{Distill}}(\theta) = \mathbb{E}_{\boldsymbol{x}_0, \boldsymbol{z}, t}\!\left[\|v_{\theta,t} - v_t\|_2^2 + \left\|u_{\theta,t} - v_t + t \cdot \frac{\mathrm{d}u_{\theta^-,t}}{\mathrm{d}t}\right\|_2^2\right]$$

第一项是标准 flow matching 损失（学速度场 v）；第二项是 consistency 项，让模型在不同 timestep 之间保持一致（这才能减少步数还不掉质量）。du/dt 是 Jacobian-vector product，用 UCGM 的二阶差分近似估计。最终在 GenEval/DPG/UniGenBench/OneIG/WISE 上分数几乎不变（GenEval 0.89→0.87，差 0.02）。

五、实验结果

不搬表格，提炼"什么动作带来什么收益"。

主结果一句话

21 个理解 benchmark 上追平 Qwen2.5-VL-7B 这种专家级 VLM（MMStar 64.1 vs 63.9，CountBench 86.0 vs 84.9 还反超）；GenEval 0.89 / DPG-Bench 87.76 / UniGenBench 79.63 在所有"统一模型"中拿第一；编辑能力 ImgEdit 3.92、MICo-Bench 47.1（比第二名 OmniGen2 33.8 高 13 分），都是 SOTA。

关键消融与启发性发现

同样的"纯离散扩散"路线：换了 VQ token 涨多少？

最具说服力的对照——和"重建型 VQ"前作 Lumina-DiMOO 同属离散扩散家族，但因为换成了语义 VQ token，在 OCR / 文档 / 图表理解这类需要细粒度语义的任务上，分数从个位数拉到 75-89。这直接证明：不是离散扩散这条路不行，是 token 选错了。

SPRINT 推理加速：1.6× 速度，0.6 分性能损失

启发性结论：免训练加速能拿到 1.6× TPS，但不同任务收益差距巨大（DocVQA 3.45× vs MathVista 1.36×）。规律：输出 token 越多的任务，per-step prefix-pruning 节省的成本累积越多。OCRBench 是唯一掉得多的（-2.3 分）——精确字符预测对 confidence 阈值更敏感。

扩散解码器蒸馏：11.4× 加速，几乎无损

其他值得记住的发现

  • WISE-Bench 推理感知生成：开"thinking" 模式让模型先 reasoning 再生成，整体分数从 0.68 → 0.78（+10 分）。统一架构天然支持 interleaved reasoning，是 LLaDA2.0-Uni 比 AR 派系最独特的优势之一。
  • 多参考编辑能力：MICo-Bench 47.1 远超 BAGEL 34.4 / OmniGen2 33.8 / Qwen-Image-Edit 35.9。同款架构的 Lumina-DiMOO 只有 23.3——再次证明语义 VQ token 在多图组合任务上的优势。
  • 多区域文字渲染稳定性：CVTG-2K 上 baselines（BAGEL、Lumina-DiMOO、InternVL-U）随 region 数增加性能急剧下降，LLaDA2.0-Uni 下降平缓。
  • 训练稳定性靠 RMSNorm-style bias 更新：MoE load balancing 的 auxiliary-loss-free bias 用 RMS 归一化（除以 RMS 强度），比直接更新平滑得多。

六、总结

  1. 核心贡献：把"VQ tokenizer 从重建型换成语义型（SigLIP-VQ）"这一刀，把纯离散扩散统一多模态模型的理解能力从 understanding-benchmark 个位数拉到 75-89 分，追平了 AR 派的 SOTA 专家 VLM——同时保持了离散扩散在并行解码、统一目标函数上的架构优势。
  2. 打开的新可能：(a) 离散扩散统一路线第一次在 understanding 上不输 AR 派，证明了纯扩散统一架构的可行性；(b) interleaved reasoning（边推理边生成）天然支持，开启了"图像版 CoT"的可能性（WISE-Bench +10 分就是证据）；(c) SigLIP-VQ 这种"语义 VQ"是个可移植组件，能给所有依赖 VQ 的下游工作（VAE-LM、image LM、video LM）都带来收益。
  3. 留下的问题：(a) SigLIP-VQ 虽然语义强，但保真度比 reconstructive VQ 差，对图像编辑等 detail-sensitive 任务还有提升空间；(b) interleaved 能力还在初步阶段，需要更大规模数据和模型；(c) RL 用于统一离散扩散 LM 还在早期探索；(d) 理解类的 reasoning benchmark（MMMU 50.1）仍落后 Lumina-DiMOO 58.6——SigLIP-VQ 的语义保留对 OCR/Chart 类好，但对深层推理可能有损。

七、Insight

论文里没直说但逻辑指向的那颗结晶。

统一多模态的瓶颈不在"用什么生成范式"，而在"中间这层离散表示有没有语义"。

过去几年统一多模态的设计辩论一直围着"AR 还是扩散"打转——AR 派指责扩散性能差、扩散派指责 AR 工程复杂——这是错的轴。LLaDA2.0-Uni 把同样的离散扩散架构（vs Lumina-DiMOO）只换了 VQ tokenizer 一个组件，在 understanding benchmark 上从个位数飙到 75-89 分。这说明性能差距的根源不在生成范式，在中间层 token 是否承载语义。

这个洞见可以推广：每当出现"两个模态用同一个 backbone 但效果差"的情况，先怀疑中间表示。AR 不是不行，扩散也不是不行，连续也好离散也好——只要中间这层 token / latent 把"语义"和"细节"恰当编码，统一就能 work。Cola DLM（用连续 latent + diffusion prior）和 LLaDA2.0-Uni（用离散语义 token + diffusion decoder）走的是两条不同的路，但内核都是这一条：把"信息层级"组织对，框架的胜负就分清了。

另一个值得抽出来记住的发现：SigLIP 这种 vision-language 对齐预训练 + VQ 量化 = 一种"自由 token"。SigLIP 提供语义对齐的连续特征，VQ 把它离散化成可以塞进 LLM 的 token。这种组合模式（pretrained semantic encoder + VQ）在所有需要"把别的模态塞进 LLM 词表"的场景都能借鉴——音频、视频、3D mesh、动作序列都可以这么干。这不是 LLaDA2.0-Uni 发明的（X-Omni 已经在用），但 LLaDA2.0-Uni 把它推到了 unified 架构的核心位置，成为论证最有力的范例。

八、启发：能动手做的事

落点是"你可以___"，不是"值得思考___"。

迁移：把"语义 VQ"思路套到任何 modality 上

如果你在做把非文本 modality 塞进 LLM 的工作（音频、3D、视频、传感器、代码 AST、分子图），SigLIP-VQ 模式可以直接套用：找一个该 modality 的 pretrained semantic encoder（CLAP for audio、PointBERT for 3D、VideoMAE for video），后接 VQ 量化成离散 token，扩展 LLM 词表，端到端训。明天可以试：选一个 audio LM 任务，把音频从 raw waveform tokenizer 换成"CLAP encoder + VQ"，看 understanding benchmark 是否大幅提升。

混搭：把它和 reasoning chain 结合

LLaDA2.0-Uni 的 WISE-Bench +10 分（开 thinking 模式）已经证明 interleaved reasoning 在统一框架里很容易加。明天可以试：在你正在做的任何"图像生成 + 推理"任务上（数学证明配图、流程图自动生成、设计稿迭代），让模型先 think → 生成 think 文字 token → 再 generate image token，全部走同一个 mask prediction 目标。这种简洁性是 AR + diffusion 混合架构难以做到的。

反转：放弃"为生成质量牺牲理解"的默认选择

很多人做统一模型时下意识地选 reconstructive VQ-VAE，因为"生成需要还原细节"。LLaDA2.0-Uni 反过来——选了语义 VQ，生成质量靠后接的 diffusion decoder 补回来，且效果更好（GenEval 0.89 是所有统一模型最高）。停下来：如果你正在为"reconstruction quality vs semantic quality"做权衡，先试试把权衡 outsource 给一个专门的下游解码器，而不是让一个 token 同时承担两件事。这条原则可能广泛适用——每当一个组件被要求做两件相互冲突的事，加个下游模块往往比硬调权重更有效。

九、关键引用（双语对照）

脱离论文上下文还能记一年的句子。

Section 2.1 · Design Principle

"Use fully discrete semantic tokens for both understanding and generation."

理解和生成共用同一套"完全离散的语义 token"——这是 LLaDA2.0-Uni 整篇论文的设计内核，一句话说尽。

Section 2.2.1 · 关于 SigLIP-VQ 的差异化

"Unlike standard VQ-VAEs that rely on pixel-level reconstruction, SigLIP-VQ is trained directly on understanding tasks, thereby preserving rich semantic information."

和标准 VQ-VAE 走"还原像素"的训练不同，SigLIP-VQ 直接在理解任务上训——这就让 token 自带语义信息。换句话说：VQ 这个工具不变，但训它的目标变了，token 的"性格"也就变了。

Section 2.2.2 · 关于 block-wise attention

"This design is particularly important for SigLIP-VQ tokens: since they are semantically aligned with Qwen2.5, they inherit an autoregressive bias that would be disrupted by pure full-attention."

block-wise attention 不是 fall-back 设计——SigLIP-VQ token 因为对齐过 Qwen2.5（一个 AR 模型），自带 AR bias，纯 full-attention 反而会破坏这个 bias 导致退化。这是个值得记住的"好设计"模式：架构选择应该跟 token 来源的 inductive bias 协调，而不是总往"理论最自由"的方向走。

Section 5.5.1 · SPRINT 启发

"The non-uniform unmasking schedule concentrates refinement on uncertain positions, effectively increasing the denoising budget for difficult tokens without additional forward passes."

SPRINT 的非均匀解 mask 把"算力预算"集中分给难 token——同样总步数，但每个难 token 实际拿到的精修次数更多。这条经验可以推广到所有迭代式生成：用 confidence 引导分配 compute，比固定 schedule 更高效。

Section 6 · Future direction

"While the SigLIP-VQ tokenizer provides rich semantic information, it struggles to preserve fine-grained image details."

作者自己点出了 SigLIP-VQ 的短板：语义强但保真弱。这预示了下一代统一模型的方向——可能是"语义 VQ + reconstructive VQ 双流"，或者更聪明的 token 设计能同时承担两种信息。后续工作的发力点。

十、Q&A

8 道面试题，覆盖结构、输入输出、训练数据、训练流程、关键策略、具体方法、实验细节、思考延伸。先想再展开。

结构Q1：LLaDA2.0-Uni 由哪几个模块组成？模块之间怎么连？
三大模块：
  • SigLIP-VQ Tokenizer：基于 SigLIP2-g ViT 的特征提取器 + vector quantizer，把任意分辨率图像编成 16,384 码本、维度 2048 的离散语义 token。文本走标准 LLM tokenizer。
  • 16B MoE 离散扩散主干（LLaDA2.0-mini）：词表扩展后包含原文本 token + SigLIP-VQ codebook + 特殊 token；处理混合 text/image token 序列；用 block-wise attention（块内双向、块间因果）；1D RoPE + size token 表示分辨率。
  • 扩散解码器（Z-Image-Base 6B）：把 backbone 输出的 SigLIP-VQ token 通过 flow matching 翻译回像素，同时做 2× 超分。蒸馏后只需 8 步推理。
连接：图像 → SigLIP-VQ → 离散 token → backbone（与文本 token 混排）→ 输出 mask 预测 → text 部分由 De-Tokenizer 解码、image 部分由 Diffusion Decoder 解码。关键：所有模态在 backbone 内都是同一种"离散 token"，不存在异构表示并存的问题。
输入输出Q2：训练时和推理时输入输出形态分别是什么？维度怎么对应？
训练时：输入是混合文本 + 图像的 token 序列（最大长度 8192，SFT 后期扩到 16384）。文本走文本 tokenizer；图像走 SigLIP-VQ → 离散 token，前面加 <height_H><width_W> size token。训练目标按 BDLM loss——按 timestep t 给随机 block 打 mask，让模型预测被 mask 的位置。

推理时：
  • 理解任务（VQA / OCR / chart QA）：输入图 + 问题文本，模型生成回答 token，文本 De-Tokenizer 解码成自然语言。注意：和 AR 不同，模型支持任意输出长度——不需要预定 mask 数量。
  • 生成任务（T2I）：输入文本 prompt，模型生成 SigLIP-VQ token 序列（按目标分辨率长度），扩散解码器把 token 翻译成像素图。8 步生成 1024×1024 用 2.90 秒。
  • 编辑任务：输入参考图 + 指令，模型在保留 reference 信息的基础上生成新的 image token。
  • Interleaved：text → image → text → image 自由交错，全部走同一个 backbone 在一次 sequence 内完成。
数据Q3：训练数据是怎么准备的？规模和构造特别在哪？
数据按理解 / 生成 / 编辑 / interleaved / reasoning 五类构造，每类都有讲究的清洗 pipeline：
  • 理解：image-caption + 自动 OCR 标注（PaddleOCR pseudo-label + Qwen3-VL 精修，避免人工）+ Grounding/Counting（Objects365、RefCOCO，coordinates 归一化到 [0,1000]）+ World knowledge & reasoning + 纯文本（Ling2.0、LLaDA2.0）。SFT 60M 样本，文本 : 多模态 = 1 : 5。
  • 生成：200M 网图原始 → 三阶段过滤（metadata + ArtiMuse aesthetic 60+ + DeQA-Score 4.0+） → 140M 高质量。captions 由 Qwen3-VL-235B 重写，保留原 caption 中的世界知识（如品牌名）。
  • 编辑：合并 7-8 个开源编辑数据集（X2Edit、OmniEdit、Nano-consistent-150k、Pico-Banana、StructVisuals、UnicEdit、CrispEdit）+ 自合成（用生成 pipeline 产编辑对）。Qwen3-VL-235B 做 instruction refinement。
  • Interleaved：从 Koala36M 视频里抽帧，duration 10-30 秒、aesthetic >4、motion >4 严格过滤，剩 6M 片段。每 5 秒抽一帧，每 sample 2-6 帧。
  • Reasoning：Flux-6M、Zebra-CoT、Weave 加起来 8M SFT 样本——专门训"先 think 后 generate"。
特别之处：(1) 数据 pipeline 几乎全自动，靠强 VLM (Qwen3-VL-235B) 做大规模质量控制和 instruction refinement；(2) 训练时还做 Image Token 预提取（pre-extraction）——所有图先用 frozen tokenizer 跑一遍，token index 存盘，训练时直接 load，节省 encoder forward 开销；(3) Data Packing 把短样本拼成定长 sequence，减少 padding 浪费。
流程Q4：训练分几个阶段？每阶段做什么？为什么这么分？
三阶段递进式：
  • Stage 0：Vision-Language Alignment（100B tokens）— 只用 image-caption + 纯文本数据，让 dLLM backbone 第一次"看懂"视觉 token。生成 res 256→512，理解 res 800×800，sequence 8192。这一阶段是建立 token-vocab 映射——backbone 的视觉 token embedding 从随机初始化开始，需要先 align。
  • Stage 1：Multi-task Pre-training（210B tokens）— 全配方上：image caption + OCR + grounding + counting + video VQA + multimodal VQA + T2I + image editing + interleaved generation。目标：让模型在理解和生成两种模式上都建立完整能力。这是规模最大的阶段。
  • Stage 2：SFT（80B tokens，分两子阶段：8k → 16k context）— 高质量数据 + reasoning-augmented 数据 + interleaved reasoning。先 8k 学基础指令跟随，再 16k 处理长 reasoning + 复杂多图编辑。Generation res 升到 1024。
为什么这么分：(a) Stage 0 必须独立——视觉 token embedding 是新初始化的，跟着完整 mix 一起训会被各种任务"拉来拉去"导致不收敛；(b) Stage 1 必须早于 SFT——pretraining 建立通用能力，SFT 只是 polish，顺序倒了会导致 overfit；(c) Stage 2 双子阶段是 context 扩展的标准做法——直接 16k 训会因为长 sequence 上的稀疏监督导致训练不稳。
策略Q5：训练里几个关键策略各解决什么问题？
  1. Block-wise Attention — 解决"理论最优 vs 实测退化"。理论上扩散 LM 应该用 full bidirectional，但实测掉点。block-wise（块内双向、块间因果）保留并行采样能力同时不破坏 SigLIP-VQ token 自带的 AR bias（来自 SigLIP2 跟 Qwen2.5 对齐）。
  2. RMSNorm-style bias 更新（auxiliary-loss-free MoE balancing） — 解决 MoE 训练中 expert load 失衡 → routing collapse。直接更新 bias 容易过冲，用 RMS 归一化（除以全局 bias 强度）平滑了更新幅度。
  3. MTRS Loss（Mask Token Reweighting） — 解决"sample 长度差异 100×"导致的 loss 偏向问题。用 1/√(mask 数) 给每个 sample 加权——长 sample 不主导梯度，短 sample 不被忽略。
  4. Complementary Masking — 解决数据效率问题。同一 clean sample x_0 同时构造 x_t 和 x'_t（互补 mask），保证每个位置在每对样本里恰好被 uncorrupted 一次，effective 信息利用率翻倍。
  5. Diffusion Decoder 三阶段训练（warmup + multi-domain + high-fidelity refinement） — 解决 decoder 既要适配 backbone 又要保留 Z-Image 预训练的两难：先冻 semantic processor 只调 decoder（warmup），再放开全部 fine-tune，最后用高质量数据 polish。
  6. Image Token 离线预提取 + Data Packing — 解决 VQ 编码的算力浪费 + 变长 sample 的 padding 浪费。前者把图先 tokenize 存盘节省训练 forward，后者把短样本拼成定长 batch 显著提升 GPU 利用率。
共同的策略哲学：每个挑战都用一个针对性的小机制解决，不堆复杂 loss——读起来像工程团队的认真总结。
方法Q6：用直觉理解一下，SPRINT 的 Sparse Prefix Retention 为什么是"modality-aware"？
回到推理时的注意力开销——每步去噪都要 attend 完整 prefix，prefix 越长 attention 越贵。SPRINT 想剪 prefix，但要剪得聪明。

关键观察：图像 token 和文本 token 的"信息密度"完全不同。
  • 图像 token 有大量空间冗余——一张 1024×1024 图压成 1024 个 token，相邻位置高度相关，剪掉 20% 几乎不损失关键信息。所以 r_img = 0.8（保留 80%，剪 20%）。
  • 文本 token 承担指令、推理链、答案——每个 token 都是关键信息，剪掉一个就可能丢上下文。所以 r_text = 1.0（全保留，不剪）。
为什么不用统一比例？早期实验肯定试过，但单一比例必然在"剪多了文本崩 vs 剪少了加速不够"之间徘徊。分模态独立设比例把这两个目标解耦：图像激进剪、文本不动，就同时拿到了两边的最优。

启发：所有"统一架构"里只要不同模态的信息密度差异大，都该考虑 modality-aware 的优化策略——不只是 KV pruning，量化、稀疏化、缓存策略都该差异化对待。
实验Q7：评测覆盖了哪些 benchmark？复现要注意什么？
评测全景（很大）：
  • 理解 21 个：MMStar / MMBench EN+CN / MME-C / HallusionBench / RealWorldQA / SimpleVQA（一般） + MMMU / MMMU-Pro / MathVista / MathVision / WeMath（推理） + ChartQA / DocVQA / InfoVQA / CharXiv / OCRBench / AI2D（OCR/chart） + CountBench / VL-RewardBench / V*（其他）
  • 生成 6 个：GenEval / DPG-Bench / OneIG-EN / UniGenBench / CVTG-2K / WISE-Bench
  • 编辑 3 个：ImgEdit-Bench / GEdit-Bench (EN+CN) / MICo-Bench
  • Interleaved：自构建 InterGen Bench (150 samples, Gemini + Qwen3-VL 评分)
Baseline 选择：(a) 专家 VLM (Qwen2.5-VL-7B、LLaDA-V) 作 understanding 上限参照；(b) 专家 T2I (FLUX.1、Seedream 3.0、Qwen-Image、Z-Image-Turbo、LongCat-Image) 作 generation 上限；(c) 同类统一模型 (BAGEL、InternVL-U、OmniGen2、HunyuanImage 3.0、NextFlow、Janus-Pro、MMaDA、Lumina-DiMOO、LLaDA-o、Emu3.5)——这是真正的对手。

复现要注意：
  1. SigLIP-VQ tokenizer 是关键——必须用对齐过的 SigLIP2 而非 raw SigLIP，否则 AR bias 缺失，block-wise attention 失效。
  2. 三阶段训练顺序不能乱，特别是 Stage 0 必须独立。
  3. Diffusion decoder 用 Z-Image-Base 而非随机初始化——预训练权重提供了像素级 prior，从零训会需要海量 T2I 数据。
  4. Image Token 离线预提取是必须的工程优化——否则训练吞吐量被 VQ encoder forward 卡死。
  5. SFT 双阶段（8k → 16k）不能跳——直接 16k 会因为稀疏监督训不稳。
  6. SPRINT 的 confidence 阈值在 OCR/document 类任务上要保守（论文用 0.93/0.95），太低会让精确字符任务掉分（OCRBench -2.3）。
思考Q8：MMMU 上 LLaDA2.0-Uni 是 50.1，反而不如同样架构的 Lumina-DiMOO 58.6——这说明什么？
这是论文里最值得琢磨的反向数据点。SigLIP-VQ 在OCR / chart / document 这类细粒度感知任务上让 LLaDA2.0-Uni 大幅领先（DocVQA 89.5 vs 7.2，差 82 分），但在 MMMU 这种跨学科深度推理任务上反而小幅落后（50.1 vs 58.6）。

可能的解释：MMMU 测的是大学科目的多选题（医学、法律、艺术、工程……），更依赖抽象世界知识 + 多步推理，而非细粒度视觉感知。SigLIP-VQ 擅长把"图里有什么"编进 token，但对"这个图在某学科语境里意味着什么"可能不如重建型 VQ 保留得多——重建型 VQ 编码更原始的视觉信号，给 backbone 留了更多"自由解读"的空间。

本质 trade-off：semantic VQ 把视觉信息预先压缩成模型容易消化的形式——这有助于通用感知任务，但也限制了 backbone 进行更深层抽象重组的余地。这是个值得记住的设计 trade-off："语义化 token"不是 free lunch，会损失部分 abstraction headroom。

对未来工作的提示：可能需要"分层 token"——浅层用 reconstructive VQ 保留原始信号，深层用 semantic VQ 提供高层提示，让 backbone 在两层信息上自由建模。或者训练 SigLIP-VQ 时加入更多 reasoning-flavored objective，让 token 自带推理友好的结构。
